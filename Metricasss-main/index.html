Definición

Las métricas de calidad del software son medidas cuantitativas que se utilizan para evaluar la calidad y el rendimiento de un software. Proporcionan información objetiva y cuantificable sobre diferentes aspectos del software, como la eficiencia, la confiabilidad, la mantenibilidad y la usabilidad.


Puntos Claves




1

Las métricas en el desarrollo de software permiten medir el desempeño y la calidad del software en diferentes aspectos. Estas métricas pueden incluir la cantidad de errores encontrados, el tiempo de respuesta del sistema, la velocidad de ejecución de un algoritmo, entre otros.

2

Las métricas son utilizadas para evaluar el progreso del proyecto y tomar decisiones informadas sobre el desarrollo del software. Estas métricas ayudan a identificar áreas de mejora, establecer metas realistas y medir el éxito del proyecto.

3

Las métricas también son útiles para comparar diferentes versiones de software, evaluar el rendimiento de los equipos de desarrollo y realizar análisis de costo-beneficio. Además, las métricas pueden ser utilizadas para entender y mejorar la experiencia del usuario, identificando patrones de uso y comportamiento del software.

Tipos de métricas

Existen diferentes tipos de metricas, aquí se mencionan las mas conocidas
Métricas de productividad
Se refieren a la cantidad de trabajo realizado en relación con los recursos invertidos, como líneas de código escritas por hora o número de errores corregidos por día.
Métricas de calidad interna
Miden la calidad del software desde una perspectiva interna, evaluando características como la modularidad, la cohesión y la reusabilidad del código.
Métricas de calidad externa
Evalúan la calidad del software desde la perspectiva del usuario final, midiendo aspectos como la usabilidad, la eficiencia y la confiabilidad.
Métricas de mantenibilidad
evalúan la facilidad de mantener y modificar el software a lo largo del tiempo, considerando factores como la complejidad del código, la documentación y la facilidad de pruebas.
Métricas de complejidad
Evalúan la complejidad del software, como la anidación de estructuras condicionales, número de caminos en una función, entre otros.
Métricas de tamaño
Se refieren al tamaño del software, como líneas de código, número de módulos, etc.

Características

Sobre las metrícas en el desarrolo

  • Objetividad

    Las métricas deben ser basadas en datos cuantitativos y no en opiniones subjetivas.

  • Relevancia

    Las métricas deben estar directamente relacionadas con los objetivos y requisitos del software.

  • Consistencia

    Las métricas deben aplicarse de manera coherente y uniforme en todo el proceso de desarrollo y mantenimiento del software

  • Accesibilidad

    Las métricas deben ser fáciles de obtener y comprender, y deben estar disponibles para todas las partes interesadas.

Prácticas



Definir metas y objetivos claros en términos de calidad del software.



Seleccionar las métricas adecuadas para medir los aspectos relevantes del software.




Establecer procedimientos para recopilar y analizar los datos necesarios para calcular las métricas.
